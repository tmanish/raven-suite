1 Prepare your HTML — dos & don'ts

Use inline-friendly CSS. Put styles in a <style> block or directly on elements — Raven Email inlines them automatically so Outlook keeps them.
Embed images as data URIs (src="data:image/png;base64,…") — they're converted to proper inline attachments that Outlook displays. Remote https:// images also work.
Use tables for layout and keep total width ≤ 640px — Outlook's renderer (Word engine) handles tables far better than modern CSS layout.
Stick to email-safe fonts: Arial, Helvetica, Georgia, Times, Verdana, Tahoma. Provide fallbacks: font-family:Arial,sans-serif.
No external stylesheets (<link rel="stylesheet">) — email clients won't fetch them. Move those rules into a <style> block first.
No relative image paths (src="images/logo.png") — there's no folder inside an email. Use data URIs, full URLs, or attach the files in step 2.
No JavaScript, forms, or video — email clients strip them. Scripts are removed automatically.
More Outlook-specific tips
Avoid flexbox, grid, and position — Outlook desktop ignores them. Use tables and align/valign.
CSS background images are unreliable in Outlook — use real <img> tags where the picture matters.
Set explicit width/height on images so the layout holds while images load (or are blocked).
Margins are flaky in Outlook — prefer padding on table cells over margin on divs.
Media queries survive — Raven Email keeps @media rules in a retained <style> block for clients that support them (mobile Outlook, Apple Mail).

Why .eml and not .msg?
.msg is a proprietary binary container that can't be reliably written outside of Outlook itself. The good news: Outlook opens .eml natively (double-click) on Windows and Mac, formatting intact — and if you ever need a true .msg, open the .eml in Outlook and Save As → .msg.
